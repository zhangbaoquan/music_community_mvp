# FC-V3-01: 导航架构重构 (Navigation Architecture Refactoring)

## 📌 变更简述
全站路由系统由 `GetX Routing` 迁移至官方推荐的 `go_router`。彻底清除了 `MainLayout` 中基于 `IndexedStack` 和强制改写 `history.replaceState` 的缝合代码，解决了 Flutter Web 端深层路由返回死锁和浏览器 URL 栈不同步的架构级顽疾。

## 📂 涉及文件
- **核心新增**:
  - `lib/core/router/app_router.dart`: 创建基于 `StatefulShellRoute` 的全局路由树。
- **架构级修改**:
  - `lib/main.dart`: 移除所有 `GetPage`，移除 `UrlSyncObserver` 和冗余的启动屏逻辑，改用 `GetMaterialApp.router` 挂载 `routerConfig`。
  - `lib/features/layout/main_layout.dart`: 移除自研的 `NavigationController`，重构侧边栏和底栏以适配 `navigationShell` 控制跳转。
- **全站重构 (37 个文件)**:
  - 编写并执行了 Python 自动替换脚本，将全站业务线（如 `ProfileView`, `AdminLayout`, `CommentsSheet` 等）中的 `Get.toNamed`、`Get.offAllNamed` 和 `Get.back` 批量安全平替为 `appRouter.push`、`appRouter.go` 和 `appRouter.pop`。

## 💡 设计决策 (Why)
- **为何放弃 GetX Router？** 
  GetX 在 Flutter Web 端的命令式路由与浏览器的原生 PopState 存在严重的抢占冲突，这是造成前进/后退键失效和丢参的罪魁祸首。
- **为何选择 `go_router` 的 `StatefulShellRoute`？**
  这是 Flutter 官方推荐的实现平级导航（Tab栏）同时保持各分支路由历史状态的唯一正解。它无需欺骗地址栏，原生支持完美的 URL 更新和分享。
- **为何保留 GetX 进行状态管理？**
  为了控制爆炸半径。剥离路由能力后，现有的 `Get.put()`、`.obs` 等依赖注入和状态管理依旧健壮，做到了“换心不换血”。

## 🚨 注意事项与遗留问题
- 所有的 `Get.snackbar` 和 `Get.dialog` 依然可以运行，因为我们在 `main.dart` 中保留了 `GetMaterialApp.router`，这使全局上下文字段仍由 GetX 提供支持。
- 因为路由的根本变化，可能存在某些隐藏的带有深层参数（如 `Article` 参数对象）的深层链接跳转逻辑需要手工测试修正。
