# 亲亲心情笔记 — 产品需求文档（PRD v2.0）

> 版本：v2.0 | 状态：规划中 | 最后更新：2026-04-06

---

## 1. 优化目标

v2.0 的核心目标是**解决 v1.0 遗留的技术债务，并横向扩展业务能力**。在保持现有功能稳定的前提下，按优先级渐进式推进。

---

## 2. 优化项（技术债务清偿）

### OPT-01 资源管道化处理（P0 — 影响线上稳定）

**问题**：用户上传的高清封面图（可达数 MB）和高码率音频文件，裸数据直传 S3 Bucket，导致存储成本高、终端加载卡顿。

**方案**：
- 前端接入图片压缩方案，严格控制上传图片分辨率安全阈值
- 实施 WebP 格式自动转换
- 音频文件进行码率压缩

**验收标准**：
- 图片上传后大小 < 500KB
- 静态带宽消耗降低 ≥ 50%
- 用户上传体验无感知延迟

---

### OPT-02 列表性能重构 — 游标分页（P0 — 影响线上稳定）

**问题**：帖子、评论拉取采用一次性 Limit 或全量加载，随数据池扩张前端内存开销暴涨。

**方案**：
- 引入游标分页（Cursor Pagination）
- 实现上拉加载更多 + 下拉刷新状态管理
- 舍弃 ListView.builder 直接满载模式

**验收标准**：
- Feed 流首次加载 ≤ 20 条，上拉追加 ≤ 20 条
- 评论区支持分页加载
- 内存占用在 1000 条数据时无明显增长

---

### OPT-03 后端数据视点下放 — Postgres RPC（P1 — 影响用户体验）

**问题**：统计类操作（总赞数、历史评论数）严重依赖前端客户端轮询多次计算拼接，存在 `.count()` 类型隐患。

**方案**：
- 编写 Supabase SQL RPC 函数，封装复杂聚合逻辑
- 前端只需调用一个 RPC，直接获取计算结果
- 将计算工作大额托付给 PostgreSQL 引擎

**验收标准**：
- 用户 Profile 页的统计数据全部由 RPC 提供
- 前端移除所有 `.count()` 客户端调用
- RPC 响应时间 < 200ms

---

### OPT-04 Web 音视频策略兼容（P1 — 影响用户体验）

**问题**：iOS Safari 等移动端浏览器严格的自动播放策略（Auto-Play Policy），导致音频无法自动播放。

**方案**：
- 优化播放预加载队列体验
- 实现用户手势触发首次播放后的自动连播
- 添加明确的播放状态 UI 提示

**验收标准**：
- iOS Safari 上首次手势触发后可连续播放
- 播放状态在 UI 上有明确反馈
- 无浏览器控制台警告

## 3. 新增功能

### FEAT-01 1v1 灵魂私信（P1）

**描述**：利用已跑通的 Supabase Realtime 技术，增加点对点的私聊窗格。从"靠内容认识你"走向"深度社交破冰"。

**功能拆解**：
- 私信对话列表页（最近聊天 + 未读计数）
- 私信详情页（实时消息 + 发送文字 + 表情）
- 推送通知接入（新消息提醒）
- 消息已读未读状态管理

**技术方案**：
- 使用 Supabase Realtime Channel 订阅私信流
- 数据表：`private_messages`（已存在基础模型）
- 使用 GetX Controller 管理对话状态

---

### FEAT-02 AI 情绪调音师（P2）

**描述**：接入大语言模型 API，当用户写完日记后，系统自动分析文字情绪向度（Sentiment），自动推荐匹配的曲库 BGM。

**功能拆解**：
- 日记完成后触发情绪分析 API
- 情绪向度分类（快乐/悲伤/平静/愤怒/期待...）
- 基于情绪标签匹配站内音乐
- "猜你喜欢" 推荐面板

**技术方案**：
- 接入 GPT-4 / 文心一言 API
- 通过 Supabase Edge Functions 或直接前端调用
- 为 songs 表增加 mood_tags 字段

---

### FEAT-03 多端 Native App 上线（P2）

**描述**：将现有 Flutter Web 业务逻辑层原封不动，适配 Native 插件池后打包生成 iOS 和 Android 应用。

**功能拆解**：
- 适配 Native 音频播放插件
- 适配 Native 图片选择/压缩插件
- 适配推送通知（极光推送等）
- App Store / Google Play 上架准备

**技术方案**：
- Flutter 跨端编译（现有代码 0 重构目标）
- 条件编译处理 Web/Native 差异
- 分离 Web 特有的 JS 互操作代码

---

### FEAT-04 个人主页深度自定义（P2）

**描述**：允许高价值核心用户上传主页顶部背景图（Banner），定义日记专栏的颜色主题，强化个人空间归属感。

**功能拆解**：
- Banner 图上传与裁剪
- 主题色选择器
- 专栏布局自定义
- 预览模式
