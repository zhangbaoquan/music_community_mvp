## 📌 核心业务拓展

### 1v1 灵魂私信
- **来源**：v2.0 遗留规划 (FEAT-01)
- **描述**：利用已跑通的 Supabase Realtime 技术，增加点对点的私聊窗格。从"靠内容认识你"走向"深度社交破冰"。
- **核心功能**：
  - 私信对话列表页（最近聊天 + 未读计数）
  - 私信详情页（实时消息 + 发送文字 + 表情）
  - 推送通知接入（新消息提醒）
  - 消息已读未读状态管理

### 个人主页深度自定义
- **来源**：v2.0 遗留规划 (FEAT-04)
- **描述**：允许高价值核心用户上传主页顶部背景图（Banner），定义日记专栏的颜色主题，强化个人空间归属感。
- **核心功能**：
  - Banner 图上传与裁剪
  - 主题色选择器
  - 专栏布局自定义
  - 自定义预览模式

## 🤖 AI 与算法探索

### AI 情绪调音师
- **来源**：v2.0 遗留规划 (FEAT-02)
- **描述**：接入大语言模型 API，当用户写完日记后，系统自动分析文字情绪向度（Sentiment），自动推荐匹配的曲库 BGM。
- **核心功能**：
  - 日记完成后触发情绪分析 API
  - 基于情绪标签（快乐/悲伤/平静/愤怒）匹配站内音乐
  - "猜你喜欢" 智能推荐面板

## 📱 跨端与平台适配

### 多端 Native App 上线
- **来源**：v2.0 遗留规划 (FEAT-03)
- **描述**：将现有 Flutter Web 业务逻辑层原封不动，适配 Native 插件池后打包生成 iOS 和 Android 原生应用。
- **核心功能**：
  - 适配 Native 音频播放、图片选择与压缩插件
  - 条件编译处理 Web/Native 差异，分离 JS 互操作代码
  - 适配 Native 推送通知（如极光推送）
  - App Store / Google Play 上架准备

## 🛠️ 工程治理与监控

### 全局工程质量与性能监控矩阵
- **来源**：v3.0 架构评审讨论
- **描述**：建立持久化的质量监控防线，防止历史技术债死灰复燃，确保 App 在真实生产环境下的稳定性。
- **核心功能**：
  1. **全端异常与性能监控 (APM)**：引入 Sentry 或 Firebase Crashlytics，自动捕获 Flutter 未处理异常及 Native 崩溃，追踪核心页面的 UI 渲染帧率与卡顿。
  2. **内存泄漏自动化巡检**：在开发与测试期接入 `leak_tracker`，自动监控 `GetxController`、流订阅及重量级 Widget 的内存释放情况。
  3. **API 请求与大流量监控**：在 Service 层封装统一拦截器，对响应耗时过长（如 >2s）或 Payload 数据过大（如 >1MB）的 Supabase 请求进行埋点警告，防止隐性 N+1 问题。
